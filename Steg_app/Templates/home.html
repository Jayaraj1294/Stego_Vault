Frequently Asked Questions

Find answers to the most common questions about Stego Vault, including its security features, encryption methods, and watermarking capabilities.

What is the overall approach for securing data using Stego Vault?

Stego Vault follows a multi-layered security approach to ensure data confidentiality, authenticity, and integrity. The process consists of the following sequential steps:

Step 1: Watermarking
The user embeds a watermark (text or image) onto the original image. This ensures authorship protection and prevents unauthorized modifications.

Step 2: Encryption
The sensitive data is encrypted using Hybrid (AES & RSA) encryption, securing it before embedding it into an image.

Step 3: Steganography
The encrypted data is embedded into the watermarked image using the Least Significant Bit (LSB) technique, ensuring covert storage.

Step 4: Tamper Detection
SHA-256 hash-based tampering detection verifies the integrity of the image, ensuring any unauthorized modifications are identified.

The embedded watermark cannot be fully extracted, adding an extra layer of security. Only the original author retains access to the unwatermarked version, preventing unauthorized removal or reversal of the watermarking process.

What encryption algorithms are used in Stego Vault?

The application implements AES (Advanced Encryption Standard) for symmetric encryption and RSA for asymmetric encryption. AES is used to encrypt text messages, while RSA encrypts the AES key for added security.

How does the steganography feature work?

Stego Vault uses the Least Significant Bit (LSB) technique for steganography. This method hides encrypted messages or watermark data within the image by modifying the least significant bits of pixel values. The changes are imperceptible to the human eye, ensuring secure and undetectable data embedding.

Can the watermark be removed or extracted?

No, once the watermark is embedded, it cannot be fully extracted. This ensures that the original image remains secure, as only the author retains access to the unwatermarked version. This provides an additional layer of security by preventing unauthorized users from reversing the watermarking process.
Furthermore, the watermark is embedded using AES and RSA encryption, ensuring that it cannot be extracted without the corresponding decryption key. This dual-layer security mechanism enhances data protection, making unauthorized access or modification significantly more difficult.

What types of watermarking are supported?

The application allows users to embed both text and image-based watermarks. Users can also choose the location of the watermark, providing flexibility for secure embedding.

How does tampering detection work?

The tampering detection feature uses SHA-256 hashing to verify image integrity. When an image is uploaded, its hash is compared with the stored hash value to detect any unauthorized modifications.
