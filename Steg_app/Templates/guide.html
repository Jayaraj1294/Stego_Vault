Application Guide

This guide provides a step-by-step overview of the features available in this application.

  • Account Management
  • Cryptography
  • Steganography
  • Tampering Detection
  • Forensic Watermarking

Account Management

The application ensures secure user authentication and account management through encryption and best security practices.

  • Register: Users create an account by providing a valid email and password. Passwords are securely hashed before storage.
  • Login: User authentication is performed using a hashed password comparison.
  • Profile Management: Users can update personal details while maintaining security measures.
  • Password Reset: Users can reset passwords through email verification.

Algorithm Used: Passwords are hashed using bcrypt for secure storage.

 Workflow:
  1. User registers by providing an email and password.
  2. Password is hashed before being stored in the database.
  3. Upon login, the entered password is hashed and compared with the stored hash.

Encryption (AES & RSA)

The encryption module secures sensitive text messages before embedding them into images using steganography.

  1. AES (Advanced Encryption Standard) Encryption
    • Uses a randomly generated AES key (256-bit) to encrypt plaintext messages.
    • AES operates in a block cipher mode to provide high security.
    • The encrypted text can be safely stored or used in the steganography process.
  2. RSA Encryption for AES Key Protection
    • The AES key itself is encrypted using RSA, ensuring only the authorized user can retrieve it..
    • The RSA public key (2048-bit) is used to encrypt the AES key before storage, while the private key is required for decryption.
 Workflow:
  1. User enters a text message for encryption.
  2. The system generates an AES key and encrypts the message.
  3. The AES key is then encrypted using RSA.
  4. The encrypted text and AES key are stored securely.

Decryption (AES & RSA)

Decryption is responsible for retrieving and decrypting messages that were previously encrypted using AES and RSA encryption techniques. It ensures that only authorized users can access the original message by verifying the correct decryption keys before processing.

Key-Based Decryption
  • The correct RSA private key is required to decrypt the AES key.
  • The decrypted AES key is then used to recover the original message.
 Workflow:
  1. The user can either manually provide the encrypted text for decryption or retrieve the ciphertext from the database using the corresponding title stored within the system.
  2. RSA decrypts the AES key.
  3. AES decrypts the message.
  4. The original message is displayed if authentication is successfull.

Steganography

Steganography enables the hiding of encrypted text within images, ensuring that sensitive information remains concealed within digital media without raising suspicion.

Least Significant Bit (LSB) Embedding:
  • The Least Significant Bit (LSB) algorithm is used to embed encrypted text within an image’s pixel values.
  • Each pixel’s LSB is modified to store portions of the encrypted text, making the changes imperceptible to the human eye.
  • The capacity of an image to store hidden data depends on its resolution and format, ensuring the embedded text does not distort the image quality.
 Workflow:
  1. User selects a watermarked image for embedding.
  2. The encrypted text is hidden within the image using LSB steganography techniques.
  3. A steganographed image is generated, ensuring it visually remains unchanged.
  4. The user downloads and stores the steganographed image, which now contains the hidden encrypted message.

Steganalysis

Steganalysis is used to detect the presence of hidden data within images, analyzing whether an image has been manipulated to contain concealed messages.

Detection Techniques
  1. Statistical Analysis: Evaluates pixel values to identify anomalies that may indicate hidden data.
  2. Histogram Deviation: Examines irregularities in color distribution that may be caused by data embedding.
  3. Noise Detection: Identifies unintended distortions or visual inconsistencies introduced by steganography.
 Workflow:
  1. User uploads an image for analysis.
  2. The system applies multiple detection algorithms to examine the image’s pixel distribution.
  3. If hidden data is found, the system extracts the data from the image.
  4. If no hidden data is detected, the image is deemed clean, ensuring no concealed messages are present.

Tampering Detection

Tampering detection ensures the integrity of an image by verifying whether it has been altered after embedding a message or watermark. This feature plays a crucial role in security applications, digital forensics, and copyright protection, helping to detect unauthorized modifications.

Key Features of Tampering Detection:
  1. Hash-Based Integrity Verification: The system calculates a unique SHA-256 hash for the original image when it is first uploaded. This hash acts as a digital fingerprint, allowing detection of even minor modifications.
  2. Real-Time Image Verification: When an image is uploaded for verification, the system recalculates its hash and compares it with the stored hash to determine if any alterations have occurred.
  3. Tampering Alerts & Reports: If a mismatch is detected, the system flags the image as tampered and alerts the user.
  4. Visual Tampering Analysis (Future Scope): Advanced implementations may include image processing techniques or AI-based methods to detect and highlight altered regions.
Algorithm Used:

SHA-256 Hashing: This cryptographic function generates a fixed-size hash value unique to each image. Even a minor pixel change results in a completely different hash, ensuring highly effective tampering detection.

 Workflow:
  1. The user uploads an image.
  2. The system calculates its SHA-256 hash and securely stores it in the database.
  3. When an image is submitted for verification, the system recalculates its SHA-256 hash.
  4. The newly computed hash is compared with the previously stored hash.
  5. Result:
    • If the hashes match, the image is considered authentic and unmodified.
    • If the hashes do not match, the system flags the image as tampered and alerts the user.
Use Cases of Tampering Detection:
  • Digital Forensics: Detects unauthorized alterations in forensic image evidence.
  • Cybersecurity & Data Protection: Prevents unauthorized modifications to sensitive images.
  • Watermark Integrity Verification: Ensures embedded watermarks remain intact and unaltered.
  • Image Authentication in Secure Applications: Confirms the originality of documents, certificates, or ID images.
 Future Enhancements:

Additional improvements may include AI-powered tampering detection, heatmap visualization of altered areas, and blockchain-based verification for immutable record-keeping.

Watermarking

Watermarking is a crucial technique used to embed identifying information into an image, ensuring authenticity, copyright protection, and security. This feature allows users to add visible or invisible watermarks, protecting digital assets from unauthorized use and modification.

Key Features of Watermarking:
  • Dual Watermarking Options: Users can embed either text or image-based watermarks.
  • Secure Encryption for Watermarks: AES and RSA encryption ensure that the embedded watermark (text or image) remains protected from unauthorized extraction.
  • Custom Watermark Positioning: The watermark placement can be selected based on user preference (e.g., top-left, top-right, center, bottom-left, bottom-right).
  • Resistant to Basic Edits: Watermarks remain embedded even after minor modifications like cropping, resizing, or compression.
Algorithm Used:

The system utilizes LSB (Least Significant Bit) Steganography for embedding watermarks, ensuring that the watermark is securely hidden while preserving image quality.

Encryption Mechanism for Watermarking:

The watermark embedding process is secured using a combination of:

  • AES (Advanced Encryption Standard): Used to encrypt the watermark (text or image) before embedding.
  • RSA (Rivest-Shamir-Adleman): Used to encrypt the AES key, ensuring that only authorized users with the private key can decrypt and extract the watermark.

This encryption mechanism prevents unauthorized extraction of the watermark, as decryption is only possible with the correct key.

 Workflow:
  1. The user selects an image to watermark.
  2. The user chooses either text or an image as the watermark.
  3. The AES key used for encryption is then encrypted with RSA and securely stored.
  4. The system applies the watermark to the image while preserving its quality.
  5. The watermarked image is generated and available for download.
 Future Enhancements:

Potential improvements include:

  • AI-Powered Invisible Watermarking: Implementing advanced techniques to hide watermarks in complex patterns, making detection harder.
  • Blockchain-Based Verification: Storing watermark verification data on the blockchain for immutable proof of ownership.
  • Robust Watermarking Techniques: Developing resistance against advanced image processing attacks, including filtering and geometric transformations.
